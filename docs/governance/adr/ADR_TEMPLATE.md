# ADR Template

```md
# ADR-###: Short Descriptive Title

Date: YYYY-MM-DD
Status: Draft | Proposed | Accepted | Deprecated | Superseded by ADR-###
Owners: Domain owner(s)

## Context

What situation, risk, or problem requires a durable decision?

## Decision

What is being decided?

## Consequences

### Positive

What improves because of this decision?

### Negative

What trade-offs or constraints does it introduce?

### Mitigations

How will the trade-offs be bounded or reduced?

## Alternatives Considered

What other options were evaluated and why were they not chosen?

## Implementation Notes

Which repository domains, database objects, APIs, UI components, or deployment steps are affected?
```
