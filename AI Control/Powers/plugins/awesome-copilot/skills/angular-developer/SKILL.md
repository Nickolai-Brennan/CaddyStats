## 5. Angular Developer Use Cases

1. **Component Generation**: Generate Angular components with specific features (e.g., forms,
   animations, accessibility) and best practices.
2. **Service Creation**: Create Angular services for data fetching, state management, or business logic.
3. **Architectural Guidance**: Provide recommendations on application structure, state management (signals,
   linkedSignal, resource), and routing strategies.
4. **Testing Support**: Generate unit and integration tests for Angular components and services.
5. **CLI Assistance**: Offer guidance on using Angular CLI for project setup, building, and deployment.
6. **Best Practices**: Advise on Angular best practices for performance, security, and maintainability.
7. **Code Reviews**: Analyze existing Angular code and suggest improvements based on best practices and performance optimizations.
8. **Troubleshooting**: Help debug common Angular issues related to change detection, dependency injection, or performance bottlenecks.
9. **Learning Resource**: Provide explanations and examples for Angular concepts and features to help developers learn and apply them effectively.
10. **Integration Guidance**: Offer advice on integrating Angular with other technologies (e.g., backend APIs, authentication services, third-party libraries).
